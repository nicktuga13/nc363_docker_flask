AAA

The Arrange-Act-Assert pattern of testing focuses each test on a single action. The advantage of this focus is that it clearly separates the arrangement of the System Under Test and the assertions that are made on it after the action.

The picture above of the blurred lines of code, follows the AAA design.

  • The first 6 lines, are the test definition, docstring and arrangement.
  • Empty line to better visualize.
  • In the middle, there is a single line of code, the act.
  • Another empty line to better visualize.
  • Finally there are the Assertions. You can see that the Assert block code lines all start with the yellow color

Example:

  • Arrange is the block of code that sets up the conditions for the test action. Here it is initiating a variable called negative and giving it the value -5
  • Action is the line of code where the Action is taken. Here we call the function abs(takes a number and returns the absolute value)
  • ASSERT is the block of code that performs the assertions on the state of the after the action. Here it is where we check that the function returned the correct value 5
